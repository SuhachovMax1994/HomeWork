Шпаргалка команд Git

  • git config --global user.name
  • git config --global user @mail
  • git init
  • git status
  • git add
  • git commit
  • git log
  • git checkout
  • git branch
  • git push
  • git pull
  • git merge
  • git clone
  • git revert
  • git reset
  • git rebase
  • git cherry-pick
  • git stash
  • Устанавливает имя, которое будет отображаться в поле автора у выполняемых вами коммитов
  • Устанавливает адрес электронной почты, который будет отображаться в информации о выполняемых вами коммитах.
  • Создаёт новый локальный репозиторий с заданным именем
  • Показывает состояние файлов (состояние: файлы изменены, состояние: файлы ждут коммита(сохранения), состояние: готовы для отправления на удаленный репозиторий.)
  • Индексирует файл для последующего коммита
  • Сохраняет проиндексированные изменения
  • Показывает историю коммитов
  • Переключает на выбранную ветку и стягивает в рабочую ветку последние обновления
  • Перечисляет все доступные ветки, так же можно удалять их и добавлять новые.
  • Загружает все измененные и созданные файлы из локального репозитория в удаленный репозиторий.
  • Стягивает файлы из удаленного репозитория в локальный. fetch(обновление) + merge(слияние)
  • Происходит слияние двух или нескольких веток в текущую.
  • Клонирует репозиторий вместе со всей его историей изменений.
  • Отмена коммита или удаление путем перезаписи новым коммитом, но при этом сохраняет историю записи предыдущего.
  • Отмена изменений
  • Команда rebase делает перебазирование текущей ветки, заменяя ее основу на, например, master.
  • Позволяет вытащить один коммит из другой ветки и вставить его в текущую ветку.
  • Своего рода черниковик. Сохраняет изменения в отдельном хранилище если нам нужно переключиться на другую задачу, при этом не вносить изменения в ветку.

Файл .gitignore хранит в себе файлы, которые будут игнорироваться Git-ом.
